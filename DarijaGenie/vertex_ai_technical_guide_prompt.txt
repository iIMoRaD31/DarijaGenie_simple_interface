Project context to describe accurately:
- The backend is a Node.js Express service located in the DarijaGenie folder.
- It uses the official `@google-cloud/vertexai` SDK.
- The service creates a `VertexAI` client with configurable project and location values.
- Environment variables used by the backend include:
  - `PORT`, defaulting to `8080`.
  - `VERTEX_PROJECT_ID`, with fallbacks to `GOOGLE_CLOUD_PROJECT`, `GCLOUD_PROJECT`, and `PROJECT_ID`.
  - `VERTEX_LOCATION`, defaulting to `global`.
  - `VERTEX_MODEL`, defaulting to `gemini-2.5-flash`.
  - `VERTEX_API_ENDPOINT`, optional, for overriding the API endpoint.
- The backend exposes:
  - `GET /health`, returning a simple health response.
  - `POST /chat`, accepting a JSON request body containing `history`.
- The `history` field is validated before calling Vertex AI.
- `history` must be an array alternating between assistant and user entries.
- Assistant entries have this shape:
  `{ "assistant": { "utterance": "...", "slots": { ... } } }`
- User entries have this shape:
  `{ "user": { "utterance": "..." } }`
- On each `/chat` request, the backend loads `prompt_5.txt`.
- `prompt_5.txt` is used as the model `systemInstruction`.
- The backend appends the new user utterance to the existing dialogue history on the client side, then sends the complete updated history to the backend.
- The backend serializes the complete `history` array and sends it to Gemini through Vertex AI as:
  `INPUT:\n<JSON.stringify(history)>\nEnd of input.`
- The backend calls `model.generateContent(requestPayload)`.
- The model output is expected to contain two parts:
  - `ASSISTANT:` with the Moroccan Darija assistant reply in Arabic script.
  - `SLOT_STATE_JSON:` with valid JSON containing tracked restaurant-order slots and an action.
- The backend parses the Gemini response, extracts the assistant text, extracts/normalizes the slot state JSON, and returns:
  `{ "assistant": "...", "slots": { ... } }`
- The backend includes defensive parsing for common model-output issues such as fenced JSON blocks, smart quotes, Python-style booleans/nulls, trailing commas, and JSON-like single quotes.
- The backend includes a Dockerfile based on Node 18, exposes port `8080`, installs dependencies, copies the app, and starts with `node server.js`.
